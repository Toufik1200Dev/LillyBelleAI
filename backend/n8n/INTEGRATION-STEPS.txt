Backend SharePoint search + n8n — steps
========================================

1) API key (backend .env)
   - Set SHAREPOINT_SEARCH_INTERNAL_KEY to a long random secret.
   - OR rely on N8N_API_KEY (same value must be sent in X-Internal-Search-Key).
   - Without one of these, POST /api/internal/sharepoint-search returns 503.

2) Start backend from backend/ folder
   npm run dev

3) Test CLI (optional)
   npm run search-metadata -- "contrat rh"

4) Test HTTP (PowerShell example — replace KEY and URL)
   $h = @{ "X-Internal-Search-Key"="KEY"; "Content-Type"="application/json" }
   $b = '{"query":"contrat rh","limit":5}' | ConvertTo-Json -Compress
   Invoke-RestMethod -Uri "http://localhost:3000/api/internal/sharepoint-search" -Method POST -Headers $h -Body '{"query":"contrat rh","limit":5}'

5) n8n HTTP Request node ("Backend Search API" / similar)
   URL:    https://<your-render-host>/api/internal/sharepoint-search
           (local: http://localhost:3000/...)
   Method: POST
   Authentication: Header Auth (credential, e.g. name "Backend Internal Key")
     - In Credentials: create "Header Auth" with
         Name:  X-Internal-Search-Key
         Value: <exact same secret as backend SHAREPOINT_SEARCH_INTERNAL_KEY or N8N_API_KEY>
     - Attach this credential to the HTTP Request node (do not put the secret in the node JSON).
   Remove any hardcoded header expression like $env.SHAREPOINT_SEARCH_INTERNAL_KEY in the node.
   Keep: Content-Type: application/json (in node headers or body options as usual).
   Body (JSON):
           { "query": "<expression to user message>", "limit": 15 }
   Chat payload from Express is usually: message, conversationId, userId, context.
   Try: {{ $json.message }} or {{ $json.body.message }} depending on Webhook options.

6) Response shape (feed to AI Agent as context only)
   success, data.hits[] with title, folder, url, lastModified, size, score, category

7) Reindex after new SharePoint export (local dev uses src/n8n-data)
   npm run search-metadata -- --reindex
   On Render: place JSON next to runtime code → dist/n8n-data/ (or N8N_DATA_DIR), then reindex
   or POST /api/internal/sharepoint-search/reindex with same X-Internal-Search-Key

8) Import test workflow
   backend/n8n/test-backend-sharepoint-search.json
   Edit URL; replace placeholder header with a Header Auth credential (same as step 5).

9) 401 Non autorisé (n8n → Render)
   - On Render, if SHAREPOINT_SEARCH_INTERNAL_KEY is set (even wrong/empty-looking), it wins
     over N8N_API_KEY. n8n must send the SAME value as that var, OR clear it on Render and use
     only N8N_API_KEY on both sides.
   - n8n credential: Header NAME must be exactly X-Internal-Search-Key (not Authorization).
   - No quotes around the secret in Render or n8n. Re-paste the key once; redeploy after env change.
   - Optional: use Authorization: Bearer <same-secret> in n8n instead (backend accepts either).
